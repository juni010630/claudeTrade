# SCREW SWEEP — v20 레짐 전수 나사조임/풀기 캠페인

**목표(사용자 /loop):** 모든 파라미터의 나사를 조이고/풀고(끄기·100%·극단 포함) 백테로 최적점 탐색.
**기준선:** v20 = `config/final_v20_dvol_perbook.yaml`. 전구간 2022-01-01~2026-04-14, $100 시작, 5bps 슬리피지 가정.
**v20 baseline (신선 풀런 = 진짜 현 라이브, `v20_baseline_loop`):** $4,816 / Sharpe **2.073** / Calmar 4.91 / MDD봉 **-29.96%** / 1065거래 / WR48.3% / PF1.895.
  - 책별: ema 1466(237) · multi 2914(215) · mean_rev 227(353) · macross 89.4(260)
  - ⚠️ 저장된 `v20_confirm`($7,774/Sh2.013/MDD-36.71%)는 **배포 전 t50 추세타깃 후보**(거래셋 동일·사이징만 큼). 배포 config는 **t40**(MDD방어). → 비교 기준 = 신선 baseline. census 헤드라인 인용 금지.
  - **풀런 시간 = real 1034s ≈ 17.2분, 14코어** → 배치당 ~10변형 병렬 = ~20분 wall-clock.

## 핵심 통찰 (왜 재실험이 재탕이 아닌가)
기존 전수튜닝(OAT·쿨다운·TP/SL·티어)의 **대부분은 v16/v17 base**(DVOL·macross 없는 2전략)에서 수행됨.
v20은 그 위에 **per-book DVOL + macross + 트리플 배분**이 얹힌 다른 레짐 → 기존 최적점이 v20에서도
유효한지 미검증. DVOL 타깃·배분 프론티어는 **post-hoc만** 봤고 정식 풀런 격자가 없음. = 진짜 새 여지.

## 방법론 가드 (HARD)
- 풀 유니버스(83종목, config 기본) · 5bps 가정 명기
- 연도별 2022/2023/2024/25-26 강건성 (특히 **2023=대출시드 생존 제약**)
- 판정: **Sharpe 비열위 + MDD + 2023 생존**. equity·MDD 동반이동=레버다이얼(엣지 아님)
- 스크리닝=전구간+연도슬라이스(analyze.py), 최종후보=독립 연도런 확정
- 도구: `run_variant.py`(프로덕션 build_engine 패리티) + `analyze.py`. 결과 parquet=`data/results/sw_*_daily.parquet`

## 비용 구조
- **sizing-only**(배분·DVOL타깃·clip·rpt): 거래셋 불변(진입/청산/슬롯은 사이징 이전 결정). 단 현 하네스는 풀런 재계산 → 풀런 병렬.
- **신호변경**(TP/SL·티어·corr·쿨다운·macross 파라미터): 거래셋 변경 → 풀런.
- 변형당 풀런 시간 = 측정중(baseline 타이밍). 병렬도 = min(코어-2). 배치 단위로 background 실행→완료신호 대기(폴링 금지).
